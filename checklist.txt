Для тестирования sms уведомления о событии из Web календаря, нам необходимо протестировать два блока функционала: 
1.	Настройка события (создание, редактирование, отмена и удаление)
2.	Отправка уведомления в установленное время
Чек-лист для тестирования первого блока:
1.	Проверка возможности добавления события (через выбор даты в календаре, через кнопку «добавить событие»)
2.	Проверка полей при заполнении (возможность ввода данных, валидация, обязательность полей)
3.	Проверка возможности редактирования события.
4.	Проверка возможности отмены события.
5.	Проверка возможности удаления события.
Чек-лист для тестирования второго блока:
1.	Проверка шаблона уведомления.
2.	Корректность отображения данных в уведомлении из события.
3.	Проверка своевременности рассылки уведомлений.
4.	Проверка уведомлений при отмене события.
5.	Проверка уведомлений при редактировании события.
